# nix-agent v0.1.0

## Simple install

Add this flake input and module to your NixOS config:

```nix
{
  inputs.nix-agent.url = "github:JEFF7712/nix-agent";

  outputs = { nixpkgs, nix-agent, ... }: {
    nixosConfigurations.my-host = nixpkgs.lib.nixosSystem {
      system = "x86_64-linux";
      modules = [
        nix-agent.nixosModules.default
        ({ ... }: {
          programs.nix-agent.enable = true;
        })
      ];
    };
  };
}
```

Rebuild:

```bash
sudo nixos-rebuild switch --flake .#my-host
```

Then point your MCP host at:

```json
{
  "mcpServers": {
    "nix-agent": {
      "command": "nix-agent",
      "args": []
    }
  }
}
```

## Highlights

- Ship `nix-agent` as a runnable stdio MCP server with a `nix-agent` console entrypoint.
- Add a companion `nix-agent` skill that teaches safe tool ordering and the `mcp-nixos` boundary.
- Package the project as a flake with `packages.default`, `apps.default`, `checks.default`, and `nixosModules.default`.
- Add CI coverage for `pytest`, `nix build .#default`, and `nix flake check`.

## What v1 includes

- Local MCP tools for inspection, patching, formatting, policy checks, dry activation, and controlled switch.
- Approval checks for sensitive paths and delete operations.
- A NixOS module that installs `nix-agent` into `environment.systemPackages`.
- Host config examples for Claude Code and OpenCode.
- README guidance for install, usage flow, limitations, and release verification.

## Important behavior

- `plan_change()` tells callers when `mcp-nixos` must be consulted first.
- `apply_change()` stops before `switch` when dry activation fails.
- `apply_change()` does not run formatters automatically; callers should run `run_formatters(changed_files)` first.
- `get_operation_result()` is still a placeholder in v1 and does not provide real async tracking.

## Known limitations

- File writes are intentionally unrestricted in this release and assume a trusted local environment.
- Secret payload writing is out of scope.
- Approval logic is intentionally simple and path-based.
- Flake checks default to the current system unless the user runs `nix flake check --all-systems`.
